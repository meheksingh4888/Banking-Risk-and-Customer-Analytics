# Banking Risk & Customer Analytics

## Business Requirements

### Objective

The objective of this project is to build a Banking Risk Customer Analytics platform capable of analyzing customer behavior, account performance, transaction activities, loan portfolios, branch performance, fraud indicators, and merchant transactions.

The solution provides business insights through SQL analytics, Python-based data analysis, and Power BI dashboards.

---

## Business Problems

Banks generate millions of transactions every day.

Business teams need answers to questions like:

- Which customers are high risk?
- Which branches perform best?
- Which loan types generate maximum revenue?
- Which transactions are failing?
- Which merchants generate maximum transaction volume?
- Which customers are likely to default?

## Project Overview

This project simulates a real-world banking analytics platform designed to help management make data-driven decisions. It integrates PostgreSQL, Python, SQL, ETL, and Power BI to analyze customer behavior, financial performance, loan risk, branch performance, and fraudulent transactions.

---

## Business Background

SecureTrust Bank is a nationwide retail and commercial bank operating over 120 branches. The bank serves millions of customers through savings accounts, current accounts, fixed deposits, loans, credit cards, internet banking, and mobile banking services.

As customer volume and transaction activity continue to grow, management requires a centralized analytics platform to monitor business performance, identify risks, detect fraud, and improve customer experience.

---

## Business Problems

The bank currently faces several operational and analytical challenges:

* Increasing loan default rates
* Fraudulent financial transactions
* Dormant customer accounts
* Uneven branch performance
* Limited customer segmentation
* Slow executive reporting
* Difficulty identifying high-value customers

---

## Project Objectives

The objectives of this project are:

* Design a normalized relational database in PostgreSQL.
* Generate realistic banking data using Python.
* Load and validate large datasets.
* Perform SQL-based business analysis.
* Build advanced analytical queries using CTEs, Window Functions, and Recursive CTEs.
* Improve performance using indexes and materialized views.
* Implement stored procedures, triggers, and transaction management.
* Develop an ETL pipeline using Python.
* Build interactive Power BI dashboards.
* Document the project professionally for GitHub.

---

## Expected Outcomes

The completed project will enable management to:

* Monitor business KPIs in real time.
* Identify high-risk loans.
* Detect suspicious transaction patterns.
* Measure branch performance.
* Segment customers based on financial behavior.
* Improve operational decision-making using analytics.
