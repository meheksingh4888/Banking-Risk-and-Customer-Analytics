# Abstract

The Banking Risk Customer Analytics project is a complete end-to-end data analytics solution developed to simulate real-world banking operations.

The project demonstrates database design, ETL processes, SQL analytics, Python-based data analysis, and interactive Power BI dashboards. It helps identify customer behavior, financial risk, loan performance, transaction trends, branch performance, and merchant activity.

The objective is to provide business stakeholders with actionable insights for informed decision-making.

# Introduction

Banks generate millions of records every day through customers, accounts, transactions, loans, and merchant activities.

Analyzing this data manually is inefficient.

This project provides an automated analytics solution capable of generating business reports and interactive dashboards for executive decision-making.

---

# Business Problem

Banks face several operational challenges:

- Identifying high-risk customers
- Monitoring loan defaults
- Tracking branch performance
- Detecting fraudulent transactions
- Understanding customer behavior
- Measuring financial performance

Without a centralized reporting system, decision-making becomes slower and less effective.

---

# Project Objectives

- Design a normalized PostgreSQL database
- Generate realistic banking datasets
- Build a robust ETL pipeline using Python
- Perform SQL analysis from basic to advanced
- Create analytical reports using Python
- Develop interactive Power BI dashboards

# Database Design

The database contains:

- Customers
- Accounts
- Transactions
- Loans
- Branches
- Merchants
- Loan Types
- Account Types

Relationships were implemented using primary keys and foreign keys to maintain referential integrity.

# ETL Process

The ETL pipeline follows these stages:

1. Extract raw datasets
2. Clean and validate data
3. Transform columns and data types
4. Load data into PostgreSQL
5. Verify record counts
6. Generate validation reports
